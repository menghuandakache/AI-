# AI 知识库管理平台架构设计文档

## 1. 项目定位

本项目定位为面向企业内部业务知识管理场景的 AI 知识库管理平台。平台围绕"知识生产、知识治理、知识检索、知识消费、反馈优化"构建完整闭环。

## 2. 总体技术选型

### 前端技术栈
- React 18 + TypeScript + Vite
- Ant Design 5 组件库
- React Router 6 路由管理
- TanStack Query 接口请求和缓存
- Axios HTTP 请求封装
- Zustand 状态管理

### 后端技术栈
- FastAPI API 框架
- Pydantic 数据校验
- SQLAlchemy 2.0 ORM
- Alembic 数据库迁移
- PostgreSQL + pgvector 数据存储
- Redis + Celery 缓存和异步任务
- Uvicorn ASGI 服务

### AI 能力选型
- Embedding 模型：bge-m3 (1024维)
- 向量数据库：PostgreSQL + pgvector
- 大模型：OpenAI-compatible API
- RAG 实现：手写 RAG Pipeline

## 4. 核心模块

1. 用户与权限模块 - 登录、JWT、角色控制
2. 知识库管理模块 - 知识库 CRUD、启停
3. 知识条目管理模块 - 知识 CRUD、发布、停用
4. 文档导入模块 - 上传、解析、切分、生成草稿
5. 向量索引模块 - bge-m3 embedding、pgvector 写入
6. 检索模块 - 关键词检索、语义检索、混合检索
7. Agent 问答模块 - RAG 问答、来源引用
8. 反馈与日志模块 - 问答日志、点赞点踩
9. 统计看板模块 - 知识统计、热门排行

## 5. 核心业务流程

### 知识生产流程
```
创建知识库 → 创建知识条目/导入文档 → 编辑知识内容 → 设置分类标签 → 发布为可用知识
```

### 知识消费流程
```
用户提问 → 系统检索知识库 → 召回相关知识 → AI 生成回答 → 展示来源 → 用户反馈
```

### RAG 检索问答链路
```
问题输入 → 生成 query embedding → pgvector 向量检索 → TopK 召回
→ 关键词检索 → 合并去重排序 → RAG Prompt → LLM 生成 → 输出答案+来源
```
