# 项目演示脚本

## 准备环境

```bash
# 1. 启动服务
docker compose up -d

# 2. 初始化数据
docker compose exec backend python scripts/seed_data.py
```

## 演示流程

### 第一步：登录系统 (1 分钟)

1. 打开浏览器访问 http://localhost:5173
2. 自动跳转到登录页面
3. 输入账号：admin / admin123
4. 点击登录，进入工作台页面
5. 展示工作台的统计卡片（知识总数、问答次数等）

**演示要点**：
- 登录页界面美观
- 登录成功后进入工作台
- 工作台有统计数据

### 第二步：创建知识库 (1 分钟)

1. 点击左侧菜单"知识库"
2. 点击"新建知识库"按钮
3. 填写信息：
   - 名称：财务报销知识库
   - 描述：公司财务报销相关制度、流程和FAQ
   - 业务域：财务
4. 点击确认，知识库列表中出现新知识库

**演示要点**：
- 创建流程简单清晰
- 列表支持查看、编辑、启停、删除操作

### 第三步：手动创建知识 (2 分钟)

1. 点击左侧菜单"知识条目"
2. 点击"新建知识"按钮
3. 填写知识信息：
   - 选择知识库：财务报销知识库
   - 标题：差旅报销材料要求
   - 分类：制度规范
   - 标签：报销、差旅
   - 正文：输入报销材料要求的内容
4. 点击保存
5. 在知识列表中找到刚创建的条目，点击"发布"

**演示要点**：
- 支持 Markdown 格式正文
- 支持分类和标签管理
- 发布后知识状态变为"可用"

### 第四步：文档导入 (2 分钟)

1. 点击左侧菜单"文档导入"
2. 选择目标知识库：财务报销知识库
3. 上传一份 PDF/Word 文档
4. 系统显示上传成功
5. 点击"开始解析"
6. 展示解析生成的知识草稿列表
7. 可以选择编辑或删除草稿
8. 点击"确认导入"

**演示要点**：
- 支持拖拽上传
- 自动解析文本内容
- 生成可编辑的知识草稿
- 异步任务不阻塞用户操作

### 第五步：创建专家 Agent (1 分钟)

1. 点击左侧菜单"专家Agent"
2. 点击"一键生成"按钮
3. 选择财务报销知识库
4. 系统自动生成 Agent 名称、描述和提示词
5. 展示生成的 Agent

**演示要点**：
- 一键生成功能方便快捷
- 自动配置回答风格和引用策略

### 第六步：Agent 对话问答 (3 分钟)

1. 在 Agent 列表中点击"问答"按钮
2. 进入聊天页面
3. 输入问题："差旅报销需要提交哪些材料？"
4. Agent 基于知识库生成回答
5. 右侧展示引用来源卡片
6. 展示知识标题和相似度
7. 再问一个知识库没有的问题："如何申请办公用品？"
8. Agent 提示"当前知识库中未找到相关内容"

**演示要点**：
- 回答基于知识库内容
- 引用来源清晰可追溯
- 无知识时不编造答案
- 支持流式输出

### 第七步：用户反馈 (1 分钟)

1. 在对话下方点击"有用"或"没用"
2. 提交反馈
3. 反馈记录会被保存

### 第八步：数据看板 (1 分钟)

1. 点击左侧菜单"数据看板"
2. 查看总览 Tab：知识统计、问答统计
3. 查看热门知识 Tab：展示引用次数最多的知识
4. 查看无答案问题 Tab：展示未能回答的问题
5. 查看最近问答 Tab：展示最近的问答记录

**演示要点**：
- 数据可视化清晰
- 支持多维度统计分析
- 为知识优化提供依据

### 第九步：知识检索 (1 分钟)

1. 点击左侧菜单"知识检索"
2. 输入搜索关键词
3. 选择混合检索模式
4. 展示检索结果：标题、内容摘要、相关度分数
5. 可以按知识库筛选

**演示要点**：
- 支持关键词、语义、混合三种检索
- 结果展示清晰

## 总耗时：约 13 分钟

## 答辩要点

1. **业务闭环**：从知识生产 → 治理 → 检索 → 消费 → 反馈，形成完整闭环
2. **RAG 链路**：展示从文档解析、文本切片、embedding 到最终问答的完整链路
3. **工程化**：分层架构（routes/services/repositories）、异步任务、数据模型设计
4. **防幻觉**：无答案拒答、严格引用来源、知识状态过滤
5. **可扩展**：embedding 模型和 LLM 可灵活替换
