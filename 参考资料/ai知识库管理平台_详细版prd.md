## 2. 业务背景

集团信息系统部负责企业内部信息系统建设，覆盖人事、财务、法务、采购、审批、职场、安全、隐私、合规等多个业务领域。随着企业组织规模扩大、业务流程复杂化和 AIGC 能力逐步落地，企业内部知识呈现出以下特点：

1. 知识来源分散：业务知识分布在制度文档、流程说明、FAQ、审批记录、飞书/IM 对话、代码仓库、接口文档、工单系统等多种载体中。
2. 知识结构不统一：不同业务线对同类问题的表达方式不同，知识颗粒度、版本、适用范围和责任人难以统一管理。
3. 知识复用效率低：员工遇到问题时往往依赖人工咨询、群聊搜索或反复询问专家，导致知识重复生产和专家答疑压力较大。
4. 知识更新不及时：制度、流程和系统能力经常调整，旧知识容易继续传播，影响业务执行准确性。
5. AI 应用缺少可信知识底座：企业内部 Agent、智能问答、流程助手等 AI 应用需要高质量、可追溯、可更新的知识库作为支撑，否则容易产生幻觉、过期回答或越权回答。

因此，需要构建 AI 知识库管理平台，将分散的企业业务知识沉淀为可管理、可检索、可消费、可评估、可持续优化的知识资产，并进一步支持专家 Agent、Skill/插件等智能服务形态。

---

## 3. 产品定位

AI 知识库管理平台是面向企业内部知识资产的一站式管理与智能服务平台。平台围绕知识生命周期，提供从知识生产、知识治理、知识检索到知识消费的完整能力。

产品定位可以概括为：

**面向企业内部复杂业务场景，构建可维护、可追溯、可智能调用的 AI 知识基础设施，让用户、专家和 Agent 共同参与知识生产与消费。**

---

## 4. 产品目标

### 4.1 业务目标

1. 提升企业内部知识复用效率，减少重复咨询和重复建设。
2. 降低新员工、跨部门协作人员对业务流程和系统规则的理解成本。
3. 为企业内部 Agent、智能问答、审批助手、流程助手等 AI 应用提供统一知识底座。
4. 建立知识持续更新机制，使高频问题、对话反馈和业务变化能够反哺知识库。
5. 支撑多业务域知识沉淀，逐步形成企业级知识资产管理体系。

### 4.2 产品目标

1. 支持知识的多方式生产，包括手动录入、文档导入、AI 自动提炼等。
2. 支持知识的组织、分类、标签、检索、状态管理和版本管理。
3. 支持基于知识库生成专家 Agent，并基于知识库内容完成问答。
4. 支持将知识能力封装为 Skill/插件，供其他 Agent 或业务系统调用。
5. 支持知识使用数据统计，包括访问量、命中率、问答调用量、高频问题等。
6. 支持知识生产闭环，即系统可从问答、反馈和使用日志中发现新知识候选项。

## 5. 用户角色与使用场景

### 5.1 用户角色

| 角色 | 说明 | 核心诉求 |
|---|---|---|
| 普通员工 | 企业内部知识消费者 | 快速查询流程、制度、系统操作、FAQ 等问题 |
| 业务知识管理员 | 负责某业务域知识库维护 | 创建、导入、编辑、审核、更新和下线知识 |
| 领域专家 | 对业务规则、流程制度具有权威解释能力 | 审核 AI 提炼内容，补充关键知识，减少重复答疑 |
| Agent 开发者 | 构建企业内部智能体或自动化工具 | 调用知识库能力，生成专家 Agent 或 Skill |
| 系统管理员 | 负责平台配置、权限、安全和稳定性 | 管理用户权限、知识空间、系统配置、日志审计 |
| 运营人员 | 关注知识使用效果和持续优化 | 查看热度、命中率、低满意问题和知识缺口 |

### 5.2 典型使用场景

#### 场景一：员工查询制度和流程

员工想了解报销流程、采购审批条件、法务合同审批要求、入职转岗流程等内容。通过专家 Agent 进行自然语言提问，系统检索相关知识并生成带来源依据的回答。

#### 场景二：知识管理员维护业务知识库

知识管理员将制度文档、FAQ 文档、流程说明导入平台，系统自动解析并提炼知识条目，管理员审核后发布为可用知识。

#### 场景三：专家审核 AI 提炼知识

系统从文档或对话中自动生成知识候选项，领域专家检查准确性、适用范围和时效性，确认后进入正式知识库。

#### 场景四：Agent 开发者创建专家 Agent

Agent 开发者选择某个知识库，如财务报销知识库、采购流程知识库、职场服务知识库，一键生成对应专家 Agent，并配置回答风格、引用策略和可访问范围。

#### 场景五：平台自动发现知识缺口

系统统计用户提问中无法回答、低满意度、高重复率的问题，生成待补充知识清单，提醒管理员和专家补充或更新知识。

## 6. 产品范围

### 6.1 MVP 范围

第一阶段应优先完成以下能力：

1. 知识库空间管理。
2. 知识条目的创建、编辑、删除、启用、停用。
3. 文档导入与基础解析。
4. 知识分类、标签、状态管理。
5. 关键词检索与语义检索。
6. 基于知识库的问答 Agent。
7. 问答结果引用知识来源。
8. 基础使用统计，包括知识访问量、问答次数、点赞/点踩。

### 6.2 加分能力范围

后续迭代可扩展以下能力：

1. AI 自动提炼知识候选项。
2. 自动发现高频问题并推荐新增知识。
3. 多模态知识录入与理解，包括图片、表格、流程图、截图。
4. 一键生成专家 Agent。
5. Skill/插件发布与调用管理。
6. 知识版本管理与变更对比。
7. 权限、合规、敏感信息检测和审计。
8. 知识质量评分与治理看板。

### 6.3 非本期范围

以下内容不作为首期强制范围：

1. 全企业级复杂知识图谱构建。
2. 完整数据血缘平台。
3. 跨语言自动翻译知识库。
4. 面向外部客户开放的知识服务。
5. 复杂流程自动执行，仅支持知识问答和知识调用。

## 7. 产品总体架构

平台整体分为五层：

1. 知识生产层：支持用户录入、文档导入、AI 提炼、对话沉淀、外部系统同步。
2. 知识治理层：支持分类、标签、状态、版本、审核、权限、质量评估。
3. 知识检索层：支持关键词检索、语义检索、标签筛选、混合检索和结果排序。
4. 知识消费层：支持专家 Agent 问答、Skill/插件调用、API 调用和知识卡片展示。
5. 数据分析层：支持知识热度、调用量、命中率、反馈、知识缺口和质量看板。

## 8. 核心业务流程

### 8.1 知识生产流程

1. 用户选择目标知识库。
2. 选择知识生产方式：
   - 手动创建知识。
   - 上传文档导入知识。
   - 从对话记录或工单记录中提炼知识。
   - 由 AI 根据已有材料生成候选知识。
3. 系统生成知识草稿。
4. 管理员或专家进行审核。
5. 审核通过后发布为可用知识。
6. 知识进入检索索引与向量索引。
7. 知识可被 Agent 或 Skill 消费。

### 8.2 知识管理流程

1. 管理员进入知识库管理页。
2. 按分类、标签、状态、创建人、更新时间筛选知识。
3. 查看知识详情，包括正文、来源、版本、标签、适用范围和历史记录。
4. 对知识进行编辑、启用、停用、删除、归档或提交审核。
5. 系统记录变更日志。
6. 修改后的知识重新进入索引更新流程。

### 8.3 知识消费流程：专家 Agent 问答

1. 用户选择某个专家 Agent。
2. 用户输入自然语言问题。
3. 系统识别问题意图和所属知识范围。
4. 检索模块从目标知识库中召回相关知识。
5. 系统对召回结果进行重排序和过滤。
6. Agent 基于检索结果生成回答。
7. 回答展示引用来源、相关知识条目和置信度提示。
8. 用户可进行点赞、点踩、追问或反馈。
9. 系统记录问答日志，用于后续知识优化。

### 8.4 知识闭环流程

1. 系统采集问答日志、用户反馈、低命中问题和高频问题。
2. AI 分析其中可能缺失、过期或表达不清的知识。
3. 生成知识候选项或知识优化建议。
4. 推送给知识管理员或领域专家审核。
5. 审核通过后更新知识库。
6. 新知识重新参与后续检索与问答。

## 9. 功能需求

## 9.1 首页与工作台

### 9.1.1 功能说明

首页用于展示用户可访问的知识库、最近使用的 Agent、待处理任务和知识统计概览。

### 9.1.2 功能点

| 功能 | 说明 | 优先级 |
|---|---|---|
| 我的知识库 | 展示当前用户有权限访问或管理的知识库 | P0 |
| 最近使用 | 展示最近访问的知识条目、Agent、Skill | P1 |
| 待处理事项 | 展示待审核知识、待处理反馈、待补充问题 | P1 |
| 数据概览 | 展示知识总数、可用知识数、问答次数、命中率 | P1 |
| 快捷入口 | 创建知识、导入文档、新建 Agent、查看统计 | P0 |

### 9.1.3 页面字段

| 字段 | 说明 |
|---|---|
| 知识库名称 | 当前知识库名称 |
| 知识数量 | 当前知识库下知识条目数量 |
| 可用状态 | 知识库是否可用于消费 |
| 最近更新 | 最近一次知识更新日期 |
| 负责人 | 知识库业务负责人 |
| 调用量 | 近 7 天或近 30 天问答/检索调用次数 |

## 9.2 知识库空间管理

### 9.2.1 功能说明

知识库空间用于承载某一业务域或主题下的知识集合，例如财务报销知识库、采购流程知识库、法务合同知识库、人事制度知识库等。

### 9.2.2 功能点

| 功能 | 说明 | 优先级 |
|---|---|---|
| 创建知识库 | 支持创建新的业务知识库 | P0 |
| 编辑知识库 | 修改名称、描述、负责人、可见范围 | P0 |
| 删除知识库 | 支持删除空知识库或归档知识库 | P1 |
| 启用/停用 | 停用后不可被 Agent 或 Skill 调用 | P0 |
| 权限配置 | 配置可查看、可编辑、可审核、可调用人员 | P1 |
| 知识库概览 | 展示知识数量、状态分布、调用数据 | P1 |

### 9.2.3 创建知识库字段

| 字段 | 类型 | 是否必填 | 说明 |
|---|---|---|---|
| 知识库名称 | 文本 | 是 | 例如财务报销知识库 |
| 知识库描述 | 多行文本 | 否 | 描述知识库适用范围 |

| 负责人 | 用户选择 | 是 | 业务负责人或管理员 |

| 消费方式 | 多选 | 否 | Agent 问答、Skill 调用、API 调用 |
| 状态 | 枚举 | 是 | 启用、停用 |

## 9.3 知识条目管理

### 9.3.1 功能说明

知识条目是平台管理知识的基本单元。每个知识条目应具备标题、正文、分类、标签、状态、来源、适用范围、更新时间等信息。

### 9.3.2 功能点

| 功能 | 说明 | 优先级 |
|---|---|---|
| 创建知识 | 支持手动创建结构化知识条目 | P0 |
| 编辑知识 | 支持编辑标题、正文、标签、分类、状态 | P0 |
| 删除知识 | 支持软删除，保留操作日志 | P0 |
| 启用/停用 | 控制知识是否可被检索和消费 | P0 |
| 复制知识 | 复制现有知识形成新条目 | P2 |
| 批量操作 | 批量启用、停用、打标签、删除 | P1 |
| 知识预览 | 预览知识在消费端的展示效果 | P1 |
| 来源追溯 | 展示知识来源文档、来源段落或人工录入来源 | P1 |
| 版本记录 | 展示历史版本和变更内容 | P2 |

### 9.3.3 知识条目字段

| 字段 | 类型 | 是否必填 | 说明 |
|---|---|---|---|
| 知识标题 | 文本 | 是 | 简洁描述知识主题 |
| 知识正文 | 富文本/Markdown | 是 | 知识核心内容 |
| 知识摘要 | 文本 | 否 | 可由 AI 自动生成 |
| 所属知识库 | 选择 | 是 | 知识归属空间 |
| 一级分类 | 选择 | 是 | 例如制度、流程、FAQ、操作指南 |
| 二级分类 | 选择 | 否 | 业务自定义 |
| 标签 | 多选 | 否 | 支持自定义标签 |
| 适用范围 | 文本/选择 | 否 | 例如中国区、全球、某业务线 |
| 生效时间 | 日期 | 否 | 知识正式生效日期 |
| 失效时间 | 日期 | 否 | 过期知识自动提醒 |
| 状态 | 枚举 | 是 | 草稿、待审核、可用、不可用、已归档 |
| 来源类型 | 枚举 | 是 | 手动录入、文档导入、AI 提炼、对话沉淀 |
| 来源链接 | 链接 | 否 | 原始文档或系统地址 |
| 创建人 | 用户 | 自动 | 系统记录 |
| 更新人 | 用户 | 自动 | 系统记录 |
| 更新时间 | 时间 | 自动 | 系统记录 |

### 9.3.4 知识状态定义

| 状态 | 含义 | 是否可检索 | 是否可消费 |
|---|---|---|---|
| 草稿 | 创建中，未提交审核 | 否 | 否 |
| 待审核 | 等待管理员或专家审核 | 否 | 否 |
| 可用 | 已发布，可被检索和 Agent 使用 | 是 | 是 |
| 不可用 | 暂时下线，不参与消费 | 可配置 | 否 |
| 已归档 | 历史知识，仅用于追溯 | 否 | 否 |
| 已删除 | 软删除状态 | 否 | 否 |

### 9.3.5 交互要求

1. 编辑知识时应支持自动保存草稿。
2. 修改已发布知识时，应生成新版本，不直接覆盖历史版本。
3. 删除知识需要二次确认。
4. 停用知识时需要填写停用原因。
5. 含有过期时间的知识，在过期前应提醒负责人。

## 9.4 文档导入与 AI 提炼

### 9.4.1 功能说明

支持用户上传文档，系统自动解析文档内容，并通过 AI 提炼为知识条目候选项，降低人工录入成本。

### 9.4.2 支持文件类型

| 文件类型 | MVP 是否支持 | 说明 |
|---|---|---|
| PDF | 是 | 支持制度、流程、规范类文档 |
| Word | 是 | 支持 doc/docx |
| Markdown | 是 | 支持技术文档 |
| Excel | P1 | 支持表格知识解析 |
| 图片 | P2 | 支持截图、流程图识别 |
| 网页链接 | P2 | 支持导入内部文档链接 |

### 9.4.3 导入流程

1. 用户选择目标知识库。
2. 上传文档或输入文档链接。
3. 系统解析文档文本、标题、章节和表格。
4. 系统按段落或章节切分内容。
5. AI 识别可沉淀知识点，生成候选知识条目。
6. 用户查看候选知识列表。
7. 用户选择保留、合并、编辑或删除候选项。
8. 提交审核或直接发布。

### 9.4.4 AI 提炼规则

AI 提炼时应优先识别以下知识类型：

1. 业务规则：如审批条件、适用范围、限制条件。
2. 操作流程：如系统操作步骤、申请流程、审批流程。
3. FAQ：如常见问题和标准答案。
4. 术语解释：如业务概念、系统名词、字段含义。
5. 注意事项：如风险提示、合规要求、例外情况。
6. 联系方式：如负责团队、支持入口、升级路径。

### 9.4.5 AI 提炼结果字段

| 字段 | 说明 |
|---|---|
| 候选标题 | AI 生成的知识标题 |
| 候选正文 | AI 提炼后的知识内容 |
| 知识类型 | 规则、流程、FAQ、术语、注意事项等 |
| 推荐标签 | AI 推荐标签 |
| 来源章节 | 原文对应章节或页码 |
| 置信度 | AI 对提炼准确性的估计 |
| 审核建议 | 是否建议人工重点检查 |

### 9.4.6 约束要求

1. AI 提炼内容必须保留来源引用，便于人工核验。
2. 置信度低的候选知识不得自动发布。
3. 涉及安全、隐私、合规、财务、法务等高风险领域的知识必须人工审核。
4. AI 不应擅自补充原文不存在的制度条款和审批规则。

### 9.5.2 分类体系

1. 制度规范。

| 新建标签 | 支持管理员或用户创建标签 | P0 |
| 标签合并 | 合并重复标签 | P2 |
| 标签推荐 | AI 根据知识内容推荐标签 | P1 |

## 9.6 知识检索

### 9.6.1 功能说明

知识检索用于帮助用户和系统快速找到相关知识。检索方式应兼顾精确匹配和语义理解。

### 9.6.2 检索方式

| 检索方式 | 说明 | 优先级 |
|---|---|---|
| 关键词搜索 | 基于标题、正文、标签进行精确搜索 | P0 |
| 语义搜索 | 基于向量召回相似知识 | P0 |
| 标签筛选 | 按标签过滤知识 | P0 |
| 分类筛选 | 按业务域和知识类型筛选 | P0 |
| 状态筛选 | 按草稿、可用、不可用等状态筛选 | P0 |
| 混合检索 | 关键词 + 语义召回 + 重排序 | P1 |
| 高级搜索 | 按创建人、更新时间、来源类型筛选 | P1 |

### 9.6.3 检索结果展示

检索结果应展示：

1. 知识标题。
2. 摘要片段。
3. 命中关键词高亮。
4. 所属知识库。
5. 分类和标签。
6. 状态。
7. 最近更新时间。
8. 来源信息。
9. 使用次数或热度。

### 9.6.4 排序策略

检索结果排序应综合考虑：

1. 文本相关度。
2. 语义相似度。
3. 知识状态。
4. 更新时间。
5. 使用热度。
6. 用户权限。
7. 业务域匹配度。

## 9.7 专家 Agent 生成与问答

### 9.7.1 功能说明

平台支持基于某个知识库或多个知识库生成专家 Agent，使用户能够通过自然语言问答消费知识。

### 9.7.2 Agent 创建方式

| 创建方式 | 说明 | 优先级 |
|---|---|---|
| 手动创建 | 用户选择知识库并配置 Agent 参数 | P0 |
| 一键生成 | 基于知识库自动生成 Agent 名称、简介和提示词 | P1 |
| 模板创建 | 使用财务、人事、法务等模板创建 Agent | P2 |

### 9.7.3 Agent 配置字段

| 字段 | 类型 | 是否必填 | 说明 |
|---|---|---|---|
| Agent 名称 | 文本 | 是 | 如财务报销助手 |
| Agent 描述 | 文本 | 是 | 说明适用场景 |
| 绑定知识库 | 多选 | 是 | 可绑定一个或多个知识库 |
| 回答风格 | 单选 | 否 | 简洁、详细、流程化、客服式 |
| 引用策略 | 单选 | 是 | 必须引用、优先引用、无需展示 |
| 无答案策略 | 单选 | 是 | 提示无匹配知识、转人工、推荐相关问题 |
| 可见范围 | 部门/用户组 | 是 | 谁可以使用该 Agent |
| 状态 | 枚举 | 是 | 启用、停用 |

### 9.7.4 问答能力要求

1. Agent 回答必须优先基于知识库内容。
2. 当知识库无相关内容时，Agent 应明确说明无法基于当前知识库回答，不应编造。
3. 回答应展示引用来源，包括知识标题、来源文档、更新时间。
4. 支持多轮追问，能够结合上下文继续检索或补充回答。
5. 支持相关问题推荐。
6. 支持用户对回答点赞、点踩和提交反馈。
7. 支持管理员查看问答日志和召回知识。

### 9.7.5 问答页面展示

问答页面至少包含：

1. Agent 名称和描述。
2. 输入框。
3. 对话内容。
4. 引用知识卡片。
5. 相关问题推荐。
6. 点赞/点踩反馈。
7. 重新生成回答。
8. 转人工或反馈入口。

### 9.7.6 回答生成规则

1. 回答中涉及制度、流程、审批条件等内容时，需要引用具体知识来源。
2. 多个知识来源存在冲突时，应提示用户存在差异，并展示不同来源及更新时间。
3. 知识来源过期或不可用时，不应作为主要依据。
4. 用户问题涉及权限外知识时，应提示无权限查看。
5. 回答应尽量结构化，例如按照结论、依据、操作步骤、注意事项展示。

## 9.8 Skill/插件生产与调用

### 9.8.1 功能说明

平台支持将知识库能力封装为 Skill 或插件，供其他 Agent 平台或业务系统调用，实现知识服务能力复用。

### 9.8.2 Skill 类型

| Skill 类型 | 说明 | 示例 |
|---|---|---|
| 知识问答 Skill | 输入问题，返回基于知识库的答案 | 查询报销规则 |
| 知识检索 Skill | 输入关键词，返回知识条目列表 | 检索采购流程 |
| FAQ 推荐 Skill | 输入场景，返回常见问题 | 入职常见问题 |
| 流程指引 Skill | 输入业务目标，返回操作步骤 | 如何发起合同审批 |

### 9.8.3 Skill 配置字段

| 字段 | 说明 |
|---|---|
| Skill 名称 | 插件名称 |
| Skill 描述 | 能力说明 |
| 绑定知识库 | 调用的知识范围 |
| 输入参数 | 如 question、keyword、user_context |
| 输出格式 | 文本、JSON、知识列表等 |
| 权限范围 | 允许调用的用户、系统或 Agent |
| 调用频控 | 每分钟/每天调用限制 |
| 状态 | 启用、停用 |

### 9.8.4 Skill 调用输出示例

```json
{
  "answer": "根据当前知识库，员工报销需要先提交费用申请并上传有效票据。",
  "sources": [
    {
      "knowledge_id": "K12345",
      "title": "员工费用报销流程说明",
      "updated_at": "2026-05-01"
    }
  ],
  "confidence": 0.86,
  "status": "answered"
}
```

## 9.9 知识使用统计与可视化

### 9.9.1 功能说明

统计知识使用情况，帮助管理员识别高价值知识、低质量知识和知识缺口。

### 9.9.2 指标体系

| 指标 | 说明 | 优先级 |
|---|---|---|
| 知识访问量 | 知识条目被查看次数 | P0 |
| 问答调用量 | 知识被 Agent 召回次数 | P0 |
| 命中率 | 用户问题成功命中知识的比例 | P1 |
| 点赞率 | 用户对回答的正向反馈比例 | P1 |
| 点踩率 | 用户对回答的负向反馈比例 | P1 |
| 无答案问题数 | Agent 无法回答的问题数量 | P1 |
| 高频问题 | 重复出现的问题聚类 | P1 |
| 热门知识 | 高频访问和高频引用知识 | P1 |
| 过期知识数 | 超过有效期或长期未更新知识 | P2 |

### 9.9.3 可视化看板

看板建议包括：

1. 知识库总体概览：知识总数、可用数、待审核数、不可用数。
2. 使用趋势：近 7 天、近 30 天检索和问答调用趋势。
3. 热门知识排行：访问量和引用量最高的知识。
4. 低满意回答列表：点踩较多或反馈较差的问答。
5. 知识缺口列表：无答案问题和高频未命中问题。
6. 过期风险提醒：即将失效或长期未更新的知识。

## 9.10 自动知识沉淀闭环

### 9.10.1 功能说明

平台从实际使用过程自动发现新知识或知识优化需求，形成从消费到生产的闭环。

### 9.10.2 触发来源

1. 高频无答案问题。
2. 用户点踩并提交原因的问题。
3. 多用户重复提问的问题。
4. 专家在对话中补充的解释。
5. 工单系统中重复出现的问题。
6. 管理员导入的新文档。

### 9.10.3 自动提炼流程

1. 系统聚类相似问题。
2. AI 判断是否存在知识缺口。
3. AI 基于对话、工单或文档生成候选知识。
4. 系统标记候选知识来源和置信度。
5. 推送给对应知识库负责人审核。
6. 审核通过后进入正式知识库。

### 9.10.4 候选知识处理动作

| 动作 | 说明 |
|---|---|
| 采纳 | 作为新知识发布 |
| 编辑后采纳 | 人工修改后发布 |
| 合并 | 合并到已有知识 |
| 忽略 | 不进入知识库 |
| 转交 | 转给其他业务负责人处理 |

## 9.11 多模态知识管理

### 9.11.1 功能说明

支持图片、表格、截图、流程图等多模态内容的录入、解析和消费。

### 9.11.2 支持内容

| 内容类型 | 能力说明 | 优先级 |
|---|---|---|
| 图片 | OCR 识别图片中文字 | P2 |
| 截图 | 识别系统截图中的字段和操作路径 | P2 |
| 表格 | 解析结构化表格内容 | P1 |
| 流程图 | 提取流程节点和流转关系 | P2 |
| 附件 | 作为知识来源或补充材料 | P1 |

### 9.11.3 消费端要求

1. Agent 回答可以引用图片或表格来源。
2. 对表格类知识，回答应尽量保留关键字段和条件。
3. 对流程图类知识，回答应转化为清晰步骤。
4. 对图片识别结果，应保留原图链接，便于用户核对。

## 9.12 权限与安全

### 9.12.1 权限模型

平台至少支持以下权限：

| 权限 | 说明 |
|---|---|
| 查看 | 可查看知识内容 |
| 检索 | 可检索知识 |
| 消费 | 可通过 Agent 或 Skill 调用知识 |
| 创建 | 可创建知识 |
| 编辑 | 可编辑知识 |
| 审核 | 可审核并发布知识 |
| 删除 | 可删除或归档知识 |
| 管理 | 可管理知识库配置和权限 |

### 9.12.2 权限控制粒度

1. 平台级权限。
2. 知识库级权限。
3. 知识条目级权限。
4. Agent 级权限。
5. Skill 调用级权限。

### 9.12.3 安全要求

1. 不同用户只能访问授权范围内的知识。
2. Agent 回答不得泄露用户无权限查看的知识内容。
3. 知识导入时应进行敏感信息检测。
4. 涉及隐私、安全、合规的知识应支持强制审核。
5. 所有关键操作应记录审计日志。
6. Skill/API 调用应支持鉴权、频控和调用日志。

## 13. 检索与问答技术方案建议

## 13.1 知识入库方案

1. 文档解析：解析 PDF、Word、Markdown 等文档内容。
2. 文本清洗：去除页眉页脚、重复空格、无效字符。
3. 内容切分：按照标题、章节、段落进行切片。
4. 元数据绑定：为切片绑定知识库、分类、标签、来源、权限等信息。
5. 向量化：对切片生成 embedding。
6. 索引构建：建立关键词索引和向量索引。
7. 状态同步：仅可用知识进入正式检索索引。

## 13.2 问答链路

1. 用户输入问题。
2. 系统进行问题改写和意图识别。
3. 根据用户权限确定可检索知识范围。
4. 采用关键词检索和向量检索召回候选知识。
5. 对候选知识进行重排序。
6. 过滤低相关度和无权限内容。
7. 将高相关知识片段输入大模型生成回答。
8. 输出回答、引用来源和置信提示。
9. 记录问答日志和反馈。

## 13.3 可靠性要求

1. 生成回答必须受到检索知识约束。
2. 无相关知识时不得强行回答。
3. 引用来源必须能够回溯到知识条目或原始文档。
4. 对冲突知识应提示冲突，而不是直接选择其中一个结论。
5. 对过期知识应降低排序权重或禁止引用。

## 16. 验收标准

## 16.1 知识管理验收

1. 用户可以成功创建、编辑、删除知识条目。
2. 用户可以设置知识状态为草稿、可用、不可用等。
3. 用户可以按关键词、标签、分类和状态检索知识。
4. 停用知识不会被 Agent 消费。
5. 删除知识不会出现在知识列表和问答引用中。
6. 知识变更记录可以被追溯。

## 16.2 文档导入验收

1. 支持上传 PDF、Word、Markdown 文档。
2. 系统可以解析文档正文。
3. 系统可以生成候选知识条目。
4. 用户可以编辑、删除、合并候选知识。
5. 确认后的候选知识可以进入知识库。

## 16.3 Agent 问答验收

1. 用户可以基于知识库创建专家 Agent。
2. Agent 可以回答知识库覆盖范围内的问题。
3. Agent 回答能够展示引用来源。
4. 无相关知识时，Agent 能明确提示无法基于知识库回答。
5. 用户可以对回答进行点赞、点踩和反馈。
6. 管理员可以查看问答日志。

## 16.4 Skill 调用验收

1. 用户可以创建绑定知识库的 Skill。
2. Skill 可以通过接口调用。
3. Skill 返回结果包含答案和来源。
4. 无权限调用时返回明确错误信息。
5. 停用 Skill 后不可继续调用。

## 16.5 数据看板验收

1. 可以查看知识数量、状态分布和使用趋势。
2. 可以查看热门知识排行。
3. 可以查看 Agent 问答次数和反馈数据。
4. 可以查看无答案问题列表。
5. 可以导出或复制关键统计数据。

## 19. 风险与应对

| 风险 | 说明 | 应对策略 |
|---|---|---|
| AI 回答幻觉 | 大模型可能生成知识库外内容 | 强制引用来源，无知识不回答 |
| 知识过期 | 旧制度或旧流程继续被调用 | 设置有效期、更新提醒、过期降权 |
| 权限泄露 | Agent 可能返回用户无权限知识 | 检索前和生成前双重权限过滤 |
| 知识质量不稳定 | AI 自动提炼可能不准确 | 候选知识人工审核，置信度标记 |
| 文档解析失败 | PDF 扫描件或复杂表格解析困难 | 提供失败提示，支持人工编辑 |
| 知识重复 | 多人重复录入相似知识 | 相似知识检测和合并建议 |
| 使用率不足 | 用户仍习惯人工咨询 | 接入常用入口，提供高频场景模板 |
| 多业务域规则冲突 | 不同部门知识存在差异 | 展示来源、适用范围和更新时间 |

## 21. 示例用户故事

### 用户故事一：知识管理员导入文档

作为财务知识管理员，我希望能够上传一份报销制度文档，系统自动解析并提炼出报销条件、审批流程、票据要求等知识条目，以便我快速审核并发布到财务知识库。

2. 系统能够提炼候选知识。
3. 候选知识保留来源段落。

### 用户故事二：员工通过 Agent 查询问题

作为普通员工，我希望直接向财务报销助手提问，了解差旅报销需要哪些材料，以便快速完成报销申请。

验收条件：

1. Agent 能返回清晰答案。
2. 答案包含所需材料列表。
3. 答案展示引用知识来源。
4. 用户可以继续追问。

### 用户故事三：系统发现知识缺口

作为知识运营人员，我希望系统能识别最近一周中多次出现但 Agent 无法回答的问题，以便我推动相关负责人补充知识。

验收条件：

1. 系统展示无答案问题列表。
2. 相似问题可以聚类。
3. 可以一键生成候选知识任务。
4. 可以分配给知识库负责人处理。

### 用户故事四：Agent 开发者创建 Skill

作为 Agent 开发者，我希望将采购知识库封装成一个问答 Skill，供采购流程助手调用，以便其他 Agent 也能复用这部分知识能力。

验收条件：

1. 可以创建 Skill。
2. 可以绑定采购知识库。
3. 可以通过接口调用。
4. 返回结果包含答案和来源。

## 22. 总结

AI 知识库管理平台的核心价值不只是存储知识，而是构建企业内部知识从生产、治理、检索到智能消费的完整闭环。通过知识库、Agent、Skill、数据看板和自动沉淀机制的结合，平台能够将分散在文档、对话、工单和业务系统中的知识转化为可复用、可追溯、可持续更新的知识资产。

首期建设应聚焦知识管理和专家 Agent 问答两条主线，确保知识能够被录入、管理、检索和消费；后续逐步增强 AI 自动提炼、多模态理解、知识热度统计、Skill 生态和知识闭环能力，最终支撑集团信息系统部在复杂业务场景下实现更高效、更可信、更智能的知识服务。
