# AI 知识库管理平台 MVP 版 PRD

## 1. 项目概述

### 1.1 产品名称

AI 知识库管理平台

### 1.2 产品定位

本产品面向企业内部业务知识管理场景，先搭建一个能够完整跑通知识生产、知识管理、知识检索和知识问答消费的轻量级平台。首期不追求功能大而全，而是优先验证核心链路是否成立：

**用户能够把知识录入平台，平台能够管理和检索知识，AI 能够基于知识库完成问答，并给出可追溯的知识来源。**

### 1.3 建设思路

当前阶段采用 MVP 思路进行建设，即先完成最小可用版本，再根据真实使用情况逐步迭代。首期重点不是覆盖所有复杂能力，而是完成以下闭环：

1. 知识可以进入平台。
2. 知识可以被管理。
3. 知识可以被检索。
4. 知识可以被 Agent 消费。
5. 用户可以对回答进行反馈。
6. 管理员可以根据使用情况继续优化知识。

---

## 2. 业务背景

集团信息系统部负责企业内部信息系统建设，覆盖人事、财务、法务、采购、审批、职场等多个领域。企业日常运作中存在大量业务知识，这些知识分散在文档、对话、流程说明、FAQ 和人工经验中，存在查找困难、复用效率低、更新不及时等问题。

随着企业内部 AI Agent 和 AIGC 应用逐步落地，需要一个统一的知识库平台作为支撑，让知识能够被沉淀、检索和智能调用。首期平台希望解决的核心问题是：

1. 业务知识缺少统一沉淀入口。
2. 用户查找知识依赖人工咨询，效率较低。
3. AI 问答缺少可信知识来源，容易产生不准确回答。
4. 知识更新和使用情况缺少基础管理手段。

---

## 3. 项目目标

### 3.1 首期目标

搭建一个全流程可运行的 AI 知识库管理平台，完成从知识创建到 AI 问答的基础闭环。

首期目标包括：

1. 支持创建知识库。
2. 支持创建、编辑、删除知识条目。
3. 支持导入文档生成知识内容。
4. 支持对知识进行分类、标签和状态管理。
5. 支持关键词检索和基础语义检索。
6. 支持基于知识库的智能问答。
7. 支持回答展示引用来源。
8. 支持用户对回答进行简单反馈。

### 3.2 非首期目标

以下能力暂不作为首期重点，仅作为后续迭代方向：

1. 多模态知识理解。
2. 复杂权限体系。
3. Skill/插件市场。
4. 自动知识闭环。
5. 知识图谱。
6. 多 Agent 协同。
7. 高级数据看板。
8. 复杂审核流。

| 角色 | 说明 | 核心诉求 |
|---|---|---|
| 普通用户 | 使用知识问答能力的员工 | 快速获得基于知识库的准确回答 |

## 5. 核心业务流程

## 5.1 知识生产流程

```text
创建知识库 → 创建知识条目/导入文档 → 编辑知识内容 → 设置分类标签 → 发布为可用知识
```

说明：

1. 管理员先创建一个知识库，例如财务报销知识库。
2. 管理员可以手动创建知识，也可以上传文档。
3. 系统解析文档内容，形成可编辑的知识草稿。
4. 管理员补充标题、分类、标签和状态。
5. 发布后，知识进入可检索和可问答状态。

## 5.2 知识管理流程

```text
进入知识库 → 查看知识列表 → 搜索/筛选知识 → 编辑/停用/删除知识 → 更新索引
```

说明：

1. 管理员可以在知识库中查看所有知识条目。
2. 可以按关键词、分类、标签、状态筛选。
3. 可以编辑知识内容。
4. 可以将知识设置为可用或不可用。
5. 不可用知识不参与问答。

## 5.3 知识消费流程

```text
用户提问 → 系统检索知识库 → 召回相关知识 → AI 生成回答 → 展示来源 → 用户反馈
```

说明：

1. 用户进入知识问答页面。
2. 输入自然语言问题。
3. 系统从指定知识库中检索相关内容。
4. AI 基于检索结果生成回答。
5. 回答中展示引用的知识来源。
6. 用户可以点赞或点踩。

## 7. 功能需求

## 7.1 知识库管理

### 7.1.1 功能说明

知识库是知识的组织空间，用于承载某一业务领域或主题下的知识内容。

### 7.1.2 首期功能

| 功能 | 说明 | 优先级 |
|---|---|---|
| 创建知识库 | 新建一个知识库 | P0 |
| 编辑知识库 | 修改名称、描述、负责人 | P0 |
| 删除知识库 | 删除无用知识库 | P1 |
| 启用/停用知识库 | 控制知识库是否可被问答使用 | P0 |
| 查看知识库列表 | 展示当前已有知识库 | P0 |

### 7.1.3 字段设计

| 字段 | 说明 |
|---|---|
| 知识库名称 | 例如财务报销知识库 |
| 知识库描述 | 说明知识库用途 |
| 所属业务域 | 人事、财务、法务、采购、审批、职场等 |
| 负责人 | 知识库维护人员 |
| 状态 | 启用/停用 |
| 创建时间 | 系统自动记录 |
| 更新时间 | 系统自动记录 |

---

## 7.2 知识条目管理

### 7.2.1 功能说明

知识条目是平台管理的最小知识单元。首期以文本知识为主，支持标题、正文、分类、标签和状态。

### 7.2.2 首期功能

| 功能 | 说明 | 优先级 |
|---|---|---|
| 创建知识 | 手动新增知识条目 | P0 |
| 编辑知识 | 修改标题、正文、分类、标签 | P0 |
| 删除知识 | 删除不需要的知识 | P0 |
| 设置状态 | 设置为可用或不可用 | P0 |
| 知识列表 | 查看某个知识库下的知识 | P0 |
| 条件筛选 | 按分类、标签、状态筛选 | P1 |

### 7.2.3 字段设计

| 字段 | 说明 |
|---|---|
| 知识标题 | 知识主题 |
| 知识正文 | 具体知识内容 |
| 所属知识库 | 当前知识归属 |
| 分类 | 制度、流程、FAQ、操作指南等 |
| 标签 | 用于检索和筛选 |
| 状态 | 可用/不可用 |
| 来源类型 | 手动录入/文档导入 |
| 来源文件 | 如果来自文档，记录文件名 |
| 创建人 | 系统自动记录 |
| 更新时间 | 系统自动记录 |

### 7.2.4 状态规则

| 状态 | 含义 | 是否可检索 | 是否可问答 |
|---|---|---|---|
| 可用 | 已发布知识 | 是 | 是 |
| 不可用 | 下线或暂不使用 | 否 | 否 |
| 草稿 | 尚未发布 | 否 | 否 |

首期不做复杂审核流，管理员可以直接发布知识。

---

## 7.3 文档导入

### 7.3.1 功能说明

文档导入用于降低知识录入成本。首期只要求完成基础文档上传、解析和入库，不追求复杂 AI 自动拆解。

### 7.3.2 支持范围

| 文件类型 | 首期是否支持 | 说明 |
|---|---|---|
| PDF | 是 | 支持文本型 PDF |
| Word | 是 | 支持 docx |
| Markdown | 可选 | 如果开发成本低可支持 |
| Excel | 暂不支持 | 后续迭代 |
| 图片 | 暂不支持 | 后续迭代 |

### 7.3.3 导入流程

```text
上传文档 → 系统解析文本 → 生成知识草稿 → 管理员编辑确认 → 发布知识
```

### 7.3.4 首期要求

1. 支持上传文档。
2. 支持提取文档中的文本内容。
3. 支持将解析结果生成一条或多条知识草稿。
4. 管理员可以编辑草稿内容。
5. 确认后发布为可用知识。

首期可以采用简单策略：

1. 按标题或章节拆分文档。
2. 如果无法识别标题，则按固定长度分段。
3. 每个分段生成一个知识草稿。

## 7.4 知识检索

### 7.4.1 功能说明

检索用于让管理员和问答系统快速找到相关知识。

### 7.4.2 首期功能

| 功能 | 说明 | 优先级 |
|---|---|---|
| 关键词检索 | 根据标题、正文、标签搜索 | P0 |
| 分类筛选 | 按知识分类过滤 | P0 |
| 标签筛选 | 按标签过滤 | P1 |
| 状态筛选 | 按可用/不可用筛选 | P0 |
| 语义检索 | 根据问题召回相似知识 | P0 |

### 7.4.3 检索结果展示

检索结果展示以下内容：

1. 知识标题。
2. 命中摘要。
3. 所属知识库。
4. 分类和标签。
5. 状态。
6. 更新时间。

## 7.5 智能问答

### 7.5.1 功能说明

智能问答是首期最核心的知识消费形式。用户通过自然语言提问，系统基于知识库检索结果生成回答。

### 7.5.2 首期功能

| 功能 | 说明 | 优先级 |
|---|---|---|
| 选择知识库问答 | 用户选择一个知识库进行提问 | P0 |
| 自然语言提问 | 支持输入问题 | P0 |
| 知识召回 | 从知识库中检索相关知识 | P0 |
| AI 生成回答 | 基于召回知识生成回答 | P0 |
| 来源引用 | 展示回答依据的知识条目 | P0 |
| 无答案提示 | 找不到相关知识时明确提示 | P0 |
| 用户反馈 | 点赞/点踩 | P1 |

### 7.5.3 问答规则

1. 回答必须基于知识库内容。
2. 如果没有检索到相关知识，应提示：当前知识库中未找到相关内容。
3. 回答需要展示引用来源。
4. 不可用知识不得参与问答。
5. 用户无权限访问的知识不得参与问答。
6. 回答应尽量结构化，优先使用条目、步骤和注意事项表达。

### 7.5.4 问答页面

页面包含：

1. 知识库选择区。
2. 对话展示区。
3. 用户输入框。
4. 回答引用来源。
5. 点赞/点踩按钮。

## 11. 技术实现建议

首期技术方案建议保持简单，优先保证可运行。

## 11.1 知识入库

1. 文档上传后解析文本。
2. 文本按章节或固定长度切分。
3. 保存原始知识条目。
4. 对知识内容生成向量。
5. 建立关键词索引和向量索引。

## 11.2 问答链路

```text
用户问题 → 向量检索/关键词检索 → 召回知识片段 → 拼接上下文 → 大模型生成回答 → 返回答案和来源
```

## 11.3 首期推荐策略

1. 检索先做简单可用，不追求复杂重排序。
2. 每次问答召回 Top 3 到 Top 5 条知识。
3. 回答必须携带知识来源。
4. 没有召回结果时直接提示无法回答。
5. 问答日志全部记录，便于后续优化。

## 13. 迭代规划

## 13.1 第一阶段：全流程跑通

目标：完成最小可用平台。

范围：

1. 知识库管理。
2. 知识条目管理。
3. 文档导入。
4. 基础检索。
5. 知识库问答。
6. 来源引用。
7. 简单反馈。

## 13.2 第二阶段：体验增强

目标：提升录入效率和问答效果。

范围：

1. AI 自动提炼知识标题和摘要。
2. 文档智能拆分。
3. 检索结果重排序。
4. 问答历史。
5. 热门知识统计。
6. 低满意回答列表。

## 13.3 第三阶段：能力扩展

目标：扩展为更完整的知识服务平台。

范围：

1. 一键生成专家 Agent。
2. Skill/插件调用。
3. 多模态知识。
4. 知识审核流。
5. 知识版本管理。
6. 自动发现知识缺口。
7. 数据看板。

---

## 14. 首期不做内容清单

为保证 MVP 快速落地，以下内容首期不做或弱化处理：

1. 不做复杂多级审批流。
2. 不做复杂组织权限，只做基础用户角色和知识库可见范围。
3. 不做多模态图片和表格深度理解。
4. 不做知识图谱。
5. 不做 Skill/插件市场。
6. 不做多 Agent 协同。
7. 不做复杂 BI 看板。
8. 不做自动发布知识，AI 生成内容必须由人确认。

---

## 15. 风险与简化应对

| 风险 | 表现 | MVP 阶段应对 |
|---|---|---|
| AI 编造答案 | 知识库没有内容但模型仍回答 | 无召回结果时直接提示无法回答 |
| 文档解析不准 | PDF 或 Word 格式复杂 | 首期只支持文本型文档，允许人工编辑 |
| 知识质量不稳定 | 管理员录入内容不规范 | 提供标题、正文、分类、标签等基础规范字段 |
| 检索效果一般 | 召回知识不准确 | 记录问答日志和反馈，后续优化检索策略 |
| 功能范围膨胀 | 首期开发周期过长 | 严格控制 MVP 范围，先跑通主流程 |

---

## 16. MVP 成功标准

首期上线后，只要满足以下条件，即可认为 MVP 验证成功：

1. 管理员可以创建知识库并录入知识。
2. 文档可以被导入并转化为知识草稿。
3. 用户可以搜索知识。
4. 用户可以通过问答页面提问。
5. AI 回答能够基于知识库内容生成。
6. 回答能够展示来源。
7. 用户反馈能够被记录。
8. 管理员可以根据反馈继续维护知识。

---

## 17. 总结

MVP 版本的 AI 知识库管理平台应聚焦一件事：**让知识能够从录入到问答完整跑通。**

因此，首期不追求复杂能力，而是优先建设知识库管理、知识条目管理、文档导入、基础检索和智能问答五个核心模块。只要能够验证知识可以被沉淀、被检索、被 AI 消费，并且回答结果可追溯，就完成了第一阶段的产品目标。

后续再基于真实使用数据逐步增强 AI 自动提炼、知识闭环、多模态、Skill、专家 Agent 和数据看板等能力。
